# Contexto do Sistema de Inscrições UAIZOUK

Esta pasta contém toda a documentação de decisões, implementações e estado atual do sistema de inscrições para eventos. É essencial para qualquer LLM ou desenvolvedor entender o contexto completo do projeto.

## Estrutura

- `README.md` - Este arquivo, visão geral
- `decisions.md` - Log de todas as decisões técnicas tomadas
- `implementation.md` - Detalhes da implementação atual
- `database-schema.md` - Esquema do banco de dados
- `api-endpoints.md` - Documentação dos endpoints da API
- `asaas-integration.md` - Integração com ASAAS para pagamentos
- `deployment-checklist.md` - Checklist para deploy em produção
- `state-machine.md` - Estado atual do sistema

## Como usar

1. **Para LLMs**: Sempre consulte esta pasta antes de fazer mudanças
2. **Para desenvolvedores**: Use como referência para entender decisões passadas
3. **Para deploy**: Siga o checklist de produção
4. **Para troubleshooting**: Consulte o state machine atual

## Importante

- Esta pasta deve ser sempre atualizada quando houver mudanças
- Decisões arquiteturais devem ser documentadas aqui
- Estado atual do sistema deve ser mantido atualizado
- Configurações de produção devem ser documentadas
